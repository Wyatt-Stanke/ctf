Complete platform outage - load balancer configuration error

Between 14:32 and 15:47 UTC, all services became unreachable due to a critical misconfiguration in the primary load balancer. An automated configuration deployment inadvertently removed routing rules for all backend services. This caused a complete site outage affecting 100% of users. The issue was identified through escalated alerts and resolved by reverting the configuration change and manually re-applying the correct rules. Post-incident review revealed a gap in pre-deployment validation checks.
