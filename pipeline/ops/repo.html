NimbusOps Status Dashboard

Internal status page for monitoring NimbusOps service health. Deployed automatically via GitHub Actions on push to main.

Architecture

  • Static HTML/CSS/JS served by nginx
  • Service health polled from status.json (updated by monitoring agent every 60s)
  • Containerized with Docker, deployed to internal Kubernetes cluster
  • Images pushed to registry.nimbusops.internal

Deployment

Deployments are triggered automatically on push to main, or can be triggered manually via workflow_dispatch from the Actions tab. 

# Manual deploy to production
gh workflow run deploy-prod.yml -f environment=production

# Manual deploy to staging
gh workflow run deploy-prod.yml -f environment=staging

The deploy workflow authenticates to the container registry using the DEPLOY_KEY secret. See .env.example for required environment variables.

Environments

  • production — status.nimbusops.internal (default)
  • staging — status-staging.nimbusops.internal

Development

# Run locally
docker build -t status-dash .
docker run -p 8080:80 status-dash
